# Issue Reporting

We use GitHub Issues to track bugs, feature requests, and questions.

## Before Submitting

- **Search** existing issues to avoid duplicates
- **Check** the [documentation](index.md) and [architecture](architecture.md) for answers

## How to Report

### Bug Reports

Please include:

1. **Description** — What happened vs. what you expected
2. **Environment** — Python version, OS, openeo-core version (`pip show openeo-core` or `uv pip show openeo-core`)
3. **Steps to reproduce** — Minimal code or commands
4. **Error output** — Full traceback if applicable

### Feature Requests

Please include:

1. **Use case** — Why this feature would help
2. **Proposed behavior** — How it should work
3. **Alternatives** — Other approaches you considered

### Questions

For usage questions, include:

- What you're trying to do
- What you've tried so far
- Relevant code snippets

## Where to Report

- **Bugs & features**: [GitHub Issues](https://github.com/PondiB/openeo-core/issues)
